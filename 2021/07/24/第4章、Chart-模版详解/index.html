🚩 写在开始之前

1⃣️ {{ }} 和 . 是什么？

相较于经典的 yaml 声明式，我们可以发现，最大的变化在于 {{ .Release.Name }}。{{ }} 就是模版调用的方法，其中加上要引用的元素，模版命令就会将其对用的值作为一个对象注入模版之中，其中 . 用于分割每个命令空间内需要搜索元素，最前面的 .表示从作用域的最顶层搜索对象。模版会从最顶层的命令空间开始搜索 . 指定的对象。

由此，{{ .Release.Name }} 就应当解读为 “从最顶层的命名空间开始搜索 Release 对象，然后在其中搜索 Name 对象“。Relase 是 Helm 的内置对象，表示发布本身。

2⃣️ 为什么要这么处理？

模版的意义在于复用。通过传入不同的值来渲染出不同的结果，才是模版存在的意义，因此针对模版我们应当坚持 “可变量化“的原则，将非通用性的字段进行变量化处理，这样才能使得我们的模版更具备泛用性，增加复用的可能。

⚠️ 注意: Release 名称的长度限制

DNS 系统限制 name 字段的长度不能超过 63 个字符，因此 Helm 对 Release 的名称是有长度限制的，Helm3 对于其长度限制为 53 个字符。

二、Helm 的内置对象列表

1、Release

该对象描述了版本发布本身，内部元素对象如下：

对象 说明
.Release.Name release 的名称
.Release.Namespace 当次版包含的命名空间
.Release.IsInstall 如果当前操作为安装，则该值会设置为 true
.Release.IsUpgrade 如果当前操作为升级或者回滚，则该值会设置为 true
.Release.Revision 此次修订的版本号，初次安装时为1，每次升级或回滚操作都会自增1
.Release.Service 用于渲染当前模版的 service，Helm 中一般为 helm

2、Files

该对象其实是一组方法，通过其可以访问 chart 中文件（如，.helmignore），内部方法对象如下：

对象 说明
.Files.Get 通过文件名获取文件内容的方法，使用方法如 .Files.Getconfig.ini，config.ini 为文件名
.Files.GetBytes 用字节数组代替字符串获取文件内容的方法。 对图片之类的文件很有用
.Files.Glob 用给定的 shell glob 模式匹配文件名，并返回一个列表
.Files.Lines 逐行读取文件内容的方法。迭代文件中每一行时很有用
.Files.AsSecrets 使用Base 64编码字符串返回文件体的方法
.Files.AsConfig 使用YAML格式返回文件体的方法

3、Capabilities

该对象提供了关于 Kubernetes 集群支持功能的信息，内部元素对象如下：

对象 说明
.Capabilities.APIVersions Kubernetes 内部使用的版本集合
.Capabilities.APIVersions.Has $version 说明集群中的版本 (如, batch/v1) 或是资源 (如, apps/v1/Deployment) 是否可用
.Capabilities.KubeVersion Kubernetes的版本号
.Capabilities.KubeVersion.Version Kubernetes的版本号
.Capabilities.KubeVersion.Major Kubernetes的主版本
.Capabilities.KubeVersion.Minor Kubernetes的次版本

4、Template

该对象包含了已经被执行的当前模版的信息，内部元素对象如下:

对象 说明
.Template.Name 当前模板的命名空间文件路径 (如， mychart/templates/configmap.yaml)
.Template.BasePath 当前chart模板目录的路径 (如，mychart/templates)

5、Values

该对象是从 values.yaml 文件，用户通过 -f 参数指定的文件以及命令行 --set 参数传入的方式注入模版的。Values 默认为空。

6、Chart

该对象表示的是 Chart.yaml 文件的内容。其中任意的有效数据都可以以诸诸如 {{ .Chart.Nmae }}-{{ .Chart.Version}} 这样的方式访问。
