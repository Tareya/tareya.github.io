第7章、项目的沟通管理和干系人管理

一、沟通管理

1、基础知识

1）沟通的概念

  • 沟通就是为确保项目信息及时且恰当地规划、收集、生产、发布、存储、检索、管理、控制、监督和最终处置项目所需的各个过程

  • 管理就是沟通，没有沟通就没有管理

  • 良好的沟通促进项目的成功

  • 沟通中最重要的不仅是你发出了什么，且要关注对方接收和理解了什么、有什么样的反馈

2）沟通的分类

1. 按严肃程度划分
  • 正式沟通
  • 非正式沟通
2. 按沟通方向划分
  • 单向沟通
  • 双向沟通
  • 横向沟通
  • 纵向沟通
3. 按沟通工具划分
  • 书面沟通
  • 口头沟通

2、沟通模型和状态

1）概念

1. 编码

把思想或想法转化为他人理解的语言

2. 信息和反馈信息

编码过程所得到的结果

3. 媒介

用来传递信息的方法

4. 噪声

干扰信息传输和理解的一切因素，如距离、新技术、缺乏背景信息等

5. 解码

把信息还原成有意义的思想或想法

2）四个部分

1. 发送方
2. 接收方
3. 媒介
4. 消息

3）五个基本状态（发收理认转）

  • 已发送
  • 已收到
  • 已理解
  • 已认可
  • 已转化

3、沟通渠道和技巧

1）沟通渠道类型

1. 文字
优点：
  • 可永久保存，容易查询
  • 节约时间，阅读速度高于语言速度
  • 读者可以根据自己的速度进行调整
  • 无地理位置需求
  • 更为精准
  • 理论上可多次无损复制传播
缺点：
  • 纯文字资料损失了大量非语言符号，不利于情感的传递
  • 对于阅读者的选择无控制力
  • 无法控制何时，以及是否被阅读
2. 语言
优点：
  • 可传递情感
  • 可同时跨地域沟通
  • 比邮件更快
  • 不需要保存传递信息的优先选择渠道
缺点：
  • 不利于建立促进个人关系
  • 无法表现肢体语言
  • 做不到文字资料的精准性，把握细节能力不足
  • 说 的速度比阅读速度相对慢
3. 混合
优点：
  • 包括语言信息和非语言信息，沟通信息含量最丰富
  • 是建立集体关系和地位最好的渠道
  • 实时性最好，可以实时的做出反应和回复
  • 对接收对象的选择有一定控制力
缺点：
  • 与语言方式相同，非文字信息速度慢
  • 利用语言沟通的时候，细节把握性不强

1. 即时性维度
2. 表达方式维度

3）沟通存在的问题

1. 语言表达能力不佳或欠缺
2. 不懂得说话的技巧或艺术
3. 未能充分传达自己的信息
4. 未能有效控制自己的情绪

4）沟通的有效性

1. 概念

所有的沟通活动最终要关注的就是有效的沟通结果

2. 要素
效果（准确性）

强调在适当的时间、适当的方式、信息被准确的发送给适当的沟通参与方（信息的接收方），并且能被正确理解

效率（及时性）

强调及时提供所需的信息

3. 提高沟通效率的原则
  • 沟通内外有别
  • 采用对方能够接受的风格
  • 非正式的沟通有助于关系的融洽
  • 扫除沟通的障碍
  • 沟通的升级原则
4. 阻碍因素
  • 沟通双方的物理距离
  • 沟通的环境因素
  • 缺乏清晰的沟通渠道
  • 复杂的组织结构
  • 复杂的技术术语
  • 有害的态度
5. 如何有效的组织项目会议
  • 明确会议的目的和期望的结果（只在确定需要时才召开会议）
  • 确定参加会议的人员
  • 在会议召开前向参加者提供会议议程（明确议题，提前分发评审资料、会议通知）
  • 使会议专业化（指定专门的主持人 —— 控制时间方向）
  • 以积极、正面的态度解决问题
  • 重视会议之后的记录（会议要形成结论、发布经审核的会议纪要）
  • 重视会议结果的告知（会议纪要要落实到责任人、时间）

4、管理过程

1）规划沟通管理

1. 概念

  • 识别和记录与干系人最有效率和效果的沟通方式

  • 基于干系人相关信息需求、可用组织资产和具体项目需求，为项目沟通活动制定恰当、有效的方法和计划

2. 关注重点

制定有效性的沟通管理计划

3. 沟通管理计划的主要内容
  • 项目干系人的沟通需要（不同的干系人的需求不同）
  • 对要发布的信息进行描述（内容、描述、详尽程度）
  • 信息接收的个人或组织
  • 传达信息所需的技术或方法（
  • 沟通效率
  • 上报过程
  • 随项目进展对沟通管理计划更新与细化的方法
  • 通用词语组
4. 输入
  • 项目管理计划

  • 干系人登记册

  • 事业环境因素
  • 组织过程资产
5. 工具和技术
  • 分析沟通需求
  • 信息传递方法的选择
6. 输出
  • 项目文件更新
  • 沟通管理计划

2）管理沟通

1. 概念

  • 促进干系人之间实现有效率和效果的沟通

  • 确保项目信息及时恰当地收集、生成、发布、存储、检索、管理、监督和最终处置

2. 关注重点

确保沟通的效果

3. 输入
  • 沟通管理计划
  • 工作绩效报告
  • 事业环境因素
  • 组织过程资产
4. 工具和技术

  • 组织过程资产更新

1. 概念

  • 随时确保所有参与者之间信息流动最优化

  • 确保满足项目及其相关方的信息需求

2.关注重点

持续改进

3. 输入

  • 项目沟通

3）信息传递方式的选择

1. 交互式沟通
优点：
  • 最有效的方法
  • 快速传递
  • 快速反馈
  • 信息量大
缺点：
  • 传递中经过层次越多，信息失真可能性越大、核实的难度越大
举例：
  • 即时通信
  • 电话
  • 视频会
2. 推式沟通
优点：
  • 持久
  • 有形
  • 可核实
缺点：
  • 效率低
  • 缺乏反馈
举例：
  • 信件
  • 报告
  • 电子邮件
  • 传真
  • 日志
  • 新闻稿
3. 拉式沟通
优点：
  • 信息意义十分明确，内涵丰富
缺点：
  • 传递距离有限，界限模糊
举例：
  • 企业内网
  • 电子在线选课
  • 经验教训数据库
  • 知识库

3）沟通的方式

1. 方式
  • 参与讨论
    • 如：头脑风暴
  • 征询方式
    • 如：问卷调查
  • 推销方式（说明）
    • 如：叙述解释
  • 叙述方式
    • 如：劝说鼓动
2. 参与和控制的关系

🚩 参与程度和控制程度成反比。

3. 如何选择
  • 已掌握足够信息，不需要进一步听取意见，选择叙述方式，其次选择推销方式
  • 掌握信息不足，没有完整成型意见，选择讨论或征询方式

4）沟通常用技巧

  • 主动倾听

  • 有效利用多种手段，尽最大可能帮助自己理解信息内容

  • 有效利用多种手段，加强团体沟通技能

  • 不回避问题，尽量了解事实真相

  • 设定目标，采取必要的追踪、验证手段确定是否达到沟通目标

  • 有多层次沟通协商能力，保证最大程度满足多方利益

  • 强大的人格魅力和信用，帮助其他人树立信心

  • 强大的表达能力，帮助他人提升士气，增加团队执行力

5）沟通管理计划

  • 通用术语表
  • 干系人的沟通需求
  • 需要沟通的信息，包括语言、格式、内容、详细程度
  • 发布信息的原因
  • 发布信息以及告知收悉或做出回应（如适用）的时限和频率
  • 负责沟通相关信息的人员
  • 负责授权保密信息发布的人员
  • 将要接收信息个人或小组
  • 传递信息的技术或方法
  • 为沟通活动分配的资源，包括时间和预算
  • 问题升级程序，用于规定下层员工无法解决问题的上报时限和上报路径
  • 随项目进展，对沟通管理计划进行更新与优化的方法
  • 项目信息流向图】工作流程（兼有授权顺序）、报告清单、会议计划
  • 沟通制约因素，通常来自特定的法律法规、技术要求和组织政策
  • 项目状态会议、项目团队会议、网络会议和电子邮件信息等的指南和模版
  • 项目所用网站和项目管理软件的使用说明

6、管理沟通 VS 控制沟通

1）共性

  • 都需要根据沟通管理计划来展开
  • 都是项目沟通管理的过程

2）联系

  • 管理沟通的输出是 项目沟通，是控制沟通的输入（项目沟通即已实际展开的沟通，如已召开的项目会议）
  • 控制沟通的输出时 变更申请，会引发重新展开规沟通管理过程或管理沟通过程

3）区别

  • 两者作用各不相同。管理沟通在执行过程组，是要展开有效的沟通；控制沟通时在监控过程组，是要检查沟通情况，及时提出变更申请

二、干系人管理

1、基础知识

1）概念

  • 识别干系人的需求，并通过沟通管理满足其需求、解决其问题的过程
  • 目前是获得更多人的支持，确保项目成功

2）项目干系人

  • 能影响项目决策、活动或结果的个人、群体或组织
  • 会受或自认为会受项目决策、活动或结果的个人、群体或组织
  • 客户、发起人、执行组织和有关公众等

2、管理过程

1）识别干系人

1. 目的

帮助项目经理建立对更干系人或干系人群体的适度关注

2. 原则

识别干系人要全面、要尽早、要迭代开展，关注变化，引导参与

3. 输入

  • 采购文件

  • 组织过程资产

5. 输出
  • 干系人登记册

2）规划干系人管理

1. 目的

2. 输入

3、技术与工具

1）干系人分析技术 —— 权利/利益方格（***）

1. 概念
  • 根据干系人的权利大小和利益对其分类
  • 指明项目需要建立的与各干系人之间的关系和种类
2. 方格解析

  • B 区的干系人

    • 项目的客户、项目的领导
    • 权利高、利益高，对项目结果关注度高 —— 重点管理，及时报告

  • A 区的干系人

    • 权利大、利益低，但对项目结果关注度低 —— 令其满意，争取支持

  • C 区的干系人

    • 项目组成员

    • 利益高，但权利低，也关注结果 —— 随时告知项目状态，保持及时沟通

  • D 区的干系人

    • 没权力，对项目的关注度低 —— 花少量精力监督管理
    • 可能发挥间接作用，态度要好一些，争取支持

3）干系人参与度评估矩阵

1. 分类
不了解型：
  • 不知道项目及其潜在影响
抵制型：
  • 知道项目及其潜在影响，但抵制项目工作或成果引发的变更
中立型：
  • 了解项目，但即不支持，也不反对
支持型：
  • 了解项目及其潜在影响，并且支持项目工作及其成果
领导型：
  • 了解项目及其潜在影响，并且积极参与以确保项目取得成功
2. 示例

干系人1 需要从不知晓到支持。重点关注获取支持

干系人2 需要从中立到支持，使其满意获取支持

干系人3 本来就支持，监督即可

4、管理干系人参与 VS 控制干系人参与

1）共性

  • 都是为了确保干系人实际参与项目的程度符合所需的参与程度
  • 都可能对干系人管理工作提出变更申请

2）联系

  • 管理干系人参与过程的输出时 问题日志，是控制干系人参与的输入
  • 在项目管理实战中，管理干系人参与和控制干系人参与两者往往无法拆开

3）区别

  • 管理干系人参与在执行过程组，控制干系人参与在监控过程组
  • 管理干系人参与，是基于对什么做法有利于提升和维持干系人参与的预期，从而采取哪些做法来实现这个预期
  • 控制干系人参与，是基于对干系人参与程度和所需程度的差距，从而采取措施使实际参与程度和预期程度相匹配
