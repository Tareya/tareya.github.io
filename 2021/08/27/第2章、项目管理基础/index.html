第2章、项目管理基础

一、项目管理基础理论

1、什么是项目？

1）概念

项目是为创建某一项独特产品、服务或成果所做的临时性（一次性）努力，有特定的环境和要求

2）特点

1. 临时性（一次性）
  • 临时性并不表示时间短
  • 项目有明确的开始时间和结束时间
2. 独特性
  • 可能有一样的产品，但不可能有一样的项目
  • 不可能有两栋完全一样的楼
3. 目的性

项目需要有明确的目的性

4. 资源约束
  • 进度、成本、质量
  • 三者相辅相成又互相制约

3）目标、约束、变量、核心

1. 三大目标
  • 时间（进度）、成本（投资）、质量
2. 四大约束

范围、质量、进度、投资

3. 五大变量

范围、质量、进度、投资、安全

4. 六大核心

范围、质量、进度、投资、安全、整体

3）项目管理过程

1. 核心知识域
  • 范围管理
  • 质量管理
  • 进度管理
  • 成本管理
  • 信息安全管理
  • 整体管理
2. 过程域
  • 可研与立项
  • 启动
  • 计划
  • 实施
  • 监控
  • 收尾
3. 伴随域
  • 变更管理
  • 沟通管理
4. 保障域
  • 人力资源管理
  • 合同管理
  • 采购管理
  • 风险管理
  • 信息与配置管理
  • 知识产权管理
  • 法律法规、标准规范和职业道德规范

4）项目和日常运作的区别

二、十大知识领域和五大过程组

1、十大知识领域

🌈 速记词：

速记词 知识点 解释说明
范 范围管理 哪些工作必须做（WBS）
进 进度管理 什么时候做（进度基准）
整 整体管理 统一并协调（战略）
狗 沟通管理 如何传递信息（组织）
子 质量管理 做到什么程度（战术指导）
成 成本管理 花多少钱做（成本基准）
人 人力资源管理 用什么人做（组织）
风 风险管理 威胁和机会（例外）
采 采购管理 买什么资源（外包人或物）
干杯 干系人管理 如何获得支持（影响干系人）

–

2、五大过程组

过程 描述 要求
启动 确定目标 千里之行，始于足下
规划 制定计划 运筹帷幄，决胜千里
执行 实施计划 言出必行，行必结果
监控 优化迭代计划 审时度势，沉着应对
收尾 交付成果，总结回顾 有始有终，如履薄冰

3、汇总表

知识领域 启动过程组 计划过程组 执行过程组 监控过程组 收尾过程组
整体管理 制定项目章程 编制项目管理计划 指导和管理项目执行 监控项目工作、整体变更控制 项目收尾
范围管理 编制范围管理计划、收集需求、范围定义、建立WBS 范围确认、范围控制
进度管理 编制进度管理计划、活动定义、活动排序、资源估算、历时估算、制定进度计划 进度控制
成本管理 编制成本管理计划、成本估算、成本预算 成本控制
质量管理 制定质量管理计划 质量保证 质量控制
人力资源管理 制定人力资源管理计划 团队组件、团队建设、团队管理
沟通管理 沟通规划 管理沟通 控制沟通
风险管理 制定风险管理计划、风险识别、风险定性分析、风险定量分析、风险应对计划 分析监控
采购管理 编制采购管理计划 实施采购 控制采购 结束采购
干系人管理 识别干系人 编制干系人管理计划 管理干系人参与 控制干系人参与
