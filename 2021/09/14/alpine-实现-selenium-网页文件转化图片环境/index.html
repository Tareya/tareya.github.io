一、背景介绍

近期公司又一个需求，是截取前端页面并生成图片，用于保存一些签名凭证等。可以说是 selenium 比较常见的使用方式了，但是对于没有接触过 selenium 的技术人员来说，就是能通过搜索引擎查询到一些方案，也不知道到底是怎么一回事，实际上，个人认为这的确更加偏向前端工程师或测试、爬虫工程师的工作，有些超出 crud boys 的工作范畴了，于是在基础环境的配置上磨蹭了几天也没有解决问题。那么借着解决并记录这次问题的机会，就大概简单的介绍下相关的知识。

二、知识讲解

1、chrome、chrome driver、selenium 简介

1）Chrome 是什么？

相信一般的技术人员或者PC用户对于 Chrome 都不会陌生，它是Google 推出的一款网页浏览器，是目前市面上使用最广泛的浏览器之一。它基于开源的 WebKit 、Blink 引擎开发（其中也包含的Google 的Gears），采用更强大的 JavaScript V8 虚拟机进行多线程的 Javascrip t执行加速，这是当前其他Web浏览器所无法实现的，所以可以说 Chrome 是母亲啊最快的 Web 浏览器。

最早的beta测试版本于 2008年9月2日发布，提供了50中语言的版本，以及对 Windows、macOS、Linux、Android、IOS的跨平台支持。

2）ChromeDriver 是什么？

🌈 ChromeDriver is the WebDriver for Chrome.

WebDriver是一个开源工具，用于在许多浏览器上自动测试webapps。它提供了导航到网页，用户输入，JavaScript执行等功能。

ChromeDriver 则是由 Google开发提供的自动化测试接口，是 Chrome 专用的 WebDriver。它是 selenium2 与 chrome 通信的桥梁，为 Chromium 实现了 WebDriver 的 JsonWireProtocol。

适用于 Android版和桌面版的 Chrome （Mac、Linux、Windows、ChromeOS）

3）Selenium 是什么？

Selenium 是 ThoughWorks 提供的一款基于浏览器的开源自动化测试工具，用于模仿用户的在浏览器的手动操作。支持的浏览器包括 IE、Chrome 和 Firefox 等。常用于测试浏览器兼容性、模仿用户行为做回归和爬虫等方面，支持自动录制动作、自动生成 .Net、Perl、Python、Ruby 和 Java 等不同语言的测试脚本。

而 Selenium 的进阶版 Selenium2 则是对浏览器提供的原生API进行封装，使其成为一套面向对象的 Selenium WebDriver API。使用这套 API 可以操控浏览器的开启、关闭，打开网页，操作界面元素，控制 Cookie，还可以操作浏览器截屏、安装插件、设置代理、配置证书等。

对于其操作需要基于浏览器厂商提供的驱动，也就是我们说的 driver。由于不同的浏览器厂商会提供的自己的驱动，比如，ChromeDriver、FirefoxDriver(xpi插件)、InternetExplorerDriver(exe) 等，都各有不同，API也并没有形成统一标准（目前都支持 JSON Wire Protocol，正逐步向 W3C 标准靠拢），所以为了方便管理，Selenium 对各个厂商的驱动都进行了封装，如 selenium-chrome-driver、selenium-firefox-driver、selenium-edge-driver等。还包括了对移动应用进行测试的AndroidDriver和iOS WebDriver，以及一个基于HtmlUnit的无界面实现HtmlUnitDriver。

通过 Python、Ruby、Java、C# 等语言可以访问 Selenium WebDriverAPI。
