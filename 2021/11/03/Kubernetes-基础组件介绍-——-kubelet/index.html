Kubernetes 基础组件介绍 —— kubelet

在 K8S 的组件中，kubelet 是在每个 Node 节点上运行的 “节点代理”。它默认监听 10250 端口，接收并执行 Master 节点发来的指令，并实际管理 Pod 以及 Pod 中的容器。每个 kubelet 进程都会在 API Server 注册所在 Node 节点的信息，其注册的表示一般为主机名(hostname)、覆盖主机名的参数或公有云驱动的特定逻辑。kubelet 会通过 cAdvisor 来监控节点和容器的资源，并定期向 Master 节点汇报该节点的资源使用情况。

1、什么是节点压力驱逐？

节点压力驱逐事实上就是 kubelet 主动终止 Pod 以回收节点资源的过程。kubelet 会监控集群节点的 CPU、内存、磁盘空间和文件系统的 inode 等资源。 当这些资源中的一个或者多个达到特定的消耗水平， kubelet 可以主动地使节点上一个或者多个 Pod 失效，以回收资源防止枯竭。

在节点压力驱逐期间，kubelet 会将所选的 Pod 的 PodPhase 设置为 Failed，这回终止 Pod。Pod 的信息残留反馈为 Evicted 状态，如下图:

K8S 中对于系统资源可以分为两种属性：

  • 可压缩资源: 例如 CPU 循环、Disk I/O、网络带宽，都是可以被限制和被回收的，对于一个 Pod 来说，可以降低这些资源的使用量而不需要去杀掉 Pod
  • 不可压缩资源: 例如内存、磁盘空间，一般来说不杀掉 Pod 就没有办法回收。

而 kubelet 回收的资源主要是内存和磁盘两种，回收的基本方式如下:

  • 磁盘回收: 主要通过删除已终止的容器和未使用的镜像
  • 内存回收: 主要通过终止正在运行的 pod

2、节点压力驱逐的机制

kubelet 会根据 驱逐信号，驱逐条件、监控间隔 来作出驱逐决定。

1）驱逐信号

驱逐信号是特定资源在特定时间点的当前状态。kubelet 使用驱逐信号，通过将信号和驱逐条件进行比较，来做出驱逐决定，驱逐条件就是节点上应该可用资源的最小量。kubelet 可使用的驱逐信号如下:

驱逐信号 取值方式
memory.available memory.available:=node.status.capacity[memory]-node.stats.memory.workingSet
nodefs.available nodefs.available:=node.stats.fs.available
nodefs.inodesFree nodefs.inodesFree:=node.stats.fs.inodesFree
imagefs.available imagefs.available:=node.stats.runtime.imagefs.available
imagefs.inodesFree imagefs.inodesFree:=node.stats.runtime.imagefs.inodesFree
pid.available pid.available:=node.stats.rlimit.maxpid-node.stats.rlimit.curproc

每个驱逐信号都支持百分比或是字面值，kubelet 采用的是信号有关的总量的百分比的方式来进行判断。

1. memory.available

memory.available 的值来自 cgroupfs，而不是像 free -m 这样的系统工具，这里系统工具在容器内实际上是起不到效果的（因为容器根本上在底层是公用的系统资源）。而 CGroup 则是将容器进行了分层次区分，每个容器都会有其唯一的 CGroup 根节点，kubelet 会判断该根节点的可用资源来计算 memory.available。

具体的计算方法如此脚本memory-available.sh。kubelet 在其计算会排除了 inactive_file（即非活动 LRU 列表上基于文件来虚拟的内存的字节数）， 因为它假定在压力下内存是可回收的。

2. nodefs

nodefs 表示节点的主文件系统，适用于本地磁盘、emptyDir、日志存储等，它关注的是本地文件系统的磁盘空间 nodefs.available 和 inode 数 nodefs.inodesFree。

3. imagefs

imagefs 表示的是容器运行时，存储容器镜像和容器可写层，是将本地文件系统根据 CGroup 分层后的文件系统，它关注的是该层文件系统的磁盘空间 imagefs.available 和 inode 数 imagefs.inodesFree。

2）驱逐条件

kubelet 的驱逐条件形式为 [eviction-signal][operator][quantity]，其中:

  • eviction-signal: 即上述的驱逐信号
  • operator: 表示关系运算符，比如大于小于等
  • quantity: 表示触发驱逐条件的指标，比如 1Gi。其表示形式要与 Kubernetes 的形式匹配，支持数值形式或百分比（10%）

使用方式如下，以节点的总内存为 10Gi 为例，我们想将触发条件设置为1 Gi，则可以将驱逐条件定义为以下两种形式中的一种:

  • memory.available<10%
  • memory.available< 1G
1. 软驱逐条件

在设置软驱逐条件时，必须指定一个宽限期，否则会报错。在超过宽限期之前，kubelet 不会立刻驱逐 Pod

在设置宽限期的同时还可以指定 Pod 终止的宽限期上限，用于在触发软驱逐时优雅的终止 Pod。当两者同时被设置，并且满足软驱逐阈值时，kubelet 会使用两者间较小的那个

软驱逐的配置参数:

  • eviction-soft: 驱逐条件列表，驱逐条件如 memory.available<1.5Gi，如果驱逐条件持续较长时间超过指定的宽限期，就会触发 Pod 驱逐
  • eviction-soft-grace-period: 驱逐宽限期列表，如 memory.available=1m30s，用于定义软驱逐在正式驱逐 Pod 前必须保存多久，与驱逐条件绑定
  • eviction-max-pod-grace-period: 在满足软驱逐条件而终止 Pod 时，使用的最大允许宽限期（单位为秒）
2. 硬驱逐条件

硬驱逐条件没有宽限期，当达到硬驱逐的阈值时，kubelet 会立即杀死 Pod 以回收资源（如内存），避免资源耗尽

硬驱逐的配置参数:

  • eviction-hard: 驱逐条件列表，驱逐条件如: ``memory.available<1Gi`

kubelet 默认的硬驱逐条件:

  • memory.available<100Mi
  • nodefs.available<10%
  • imagefs.available<15%
  • nodefs.inodesFree<5%（Linux 节点）

3）驱逐监控间隔

kubelet 根据其配置的 housekeeping-interval（默认为 10s）评估驱逐条件

3、节点压力状况

kubelet 会报告节点状况以反映节点处于压力之下，其与驱逐信号的对应关系如下:

节点压力表现 驱逐信号 描述
MemoryPressure memory.available 节点上的可用内存已满足驱逐条件
DiskPressure nodefs.available、nodefs.inodesFree、imagefs.available或imagefs.inodesFree 节点的根文件系统或镜像文件系统的磁盘空间或 inode 已满足驱逐条件
PIDPressure pid.available (Linux) 节点上的可用进程标识符已低于驱逐条件

通过配置 kubelet 的参数 nodeStatusUpdateFrequency 可以调整节点压力条件的更新间隔，默认为 10s

4、节点磁盘资源回收

当报告 DiskPressure 节点压力状况时，kubelet 会根据节点上的文件系统回收节点资源

1）有 imagefs

如果节点有专用的 imagefs 文件系统供容器运行时使用，如 cgroup，kubelet 会执行以下操作:

  • 如果 nodefs 文件系统满足驱逐条件，kubelet 会对已死亡的 Pod 和容器进行垃圾回收
  • 如果 imagefs 文件系统满足驱逐条件，kubelet 会删除节点上所有未使用的容器镜像

2）没有 imagefs

如果节点只有一个满足驱逐条件的 nodefs 文件系统，kubelet 会按以下顺序释放磁盘空间

  • 对已死亡的 Pod 和容器进行垃圾回收
  • 删除未使用的容器镜像

5、节点内存资源回收

正常情况下，kubelet 会通过驱逐条件杀死 Pod 来回收内存资源，但是如果在此之前出现 OOM，Node 节点就会依赖 oom_killer 来响应。

kubelet 根据 Pod 的服务质量（QoS）会为每个容器设置一个 oom_score_adj 值
