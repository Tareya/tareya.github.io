Kubernetes Etcd 集群备份恢复方案

Etcd 是 Kubernetes 集群中极为重要的一块服务，其中存储了 Kubernetes 集群所有的数据信息，如果 Etcd 集群发生灾难或数据丢失，集群数据的恢复将不可逆，所以做好这块的备份恢复是极为重要的。

Kubernetes Etcd 集群备份恢复方案

🌈 Etcd 可以说是 Kubernetes 整个架构体系中最为重要的服务，其中存储了 Kubernetes 集群所有的数据信息，如 Namespace、Pod、Service、路由等状态信息，且 Kubernetes 体系中最重要的 list-watch 机制也是依赖 Etcd 实现的。

一、Etcd 集群备份

🚩 etcd 的版本不同，etcdctl 的命令可能会不一样，但是大致来说还是差不多的，我们这里使用 napshot save 快照的方式进行备份。需要注意以下几点:

  • 备份操作在 etcd 集群中的任何一个节点执行就可以了
  • 从 1.13 版本开始，K8S 就改用 v3 版本的 API 了，v2 和 v3 存储的路径是不同的，所以我们在备份时使用 v3 版本的 API 进行操作
  • 本案例使用的是二进制包部署的 v1.15.5 + flannel 网络的 K8S 集群
  • 下面命令中的 ETCDCTL_API=3 etcdctl 等效于 etcdctl，通过变量的方式同时 etcd 使用 v3 版本 API

1、检查 etcd 的数据

1）etcd 数据目录结构

路径 描述
/data/application/etcd/ etcd 集群程序目录（企业目录标准）
default.etcd/ etcd 默认数据目录
default.etcd/member/snap etcd 默认状态快照存储目录，目的是为了避免 wal 文件过多
default.etcd/member/wal etcd 默认 wal（write ahead log）日志存储目录，wal 相当于 etcd 的 redo log，是集群保持一致的关键
